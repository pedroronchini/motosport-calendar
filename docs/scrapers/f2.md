# Scraper: FIA Formula 2

> Pré-requisitos: ver `00-arquitetura-e-schema.md`.
> **Status: IMPLEMENTADO** (`app/Services/Scrapers/FiaFormulaScraper.php` +
> `F2Scraper.php`, comando `scrape:f2`). Ver seção "Fonte de dados (validada)".

## Visão geral

Categoria de acesso à F1, corre em **suporte à F1** em ~14 rounds. Um
`championship` (`slug = f2`, `category = single_seater`). **Os circuitos são os
mesmos da F1** — reutilizar os registros já existentes casando por `slug`.

## Fonte de dados (validada em jul/2026)

**Jolpica/Ergast NÃO cobre F2/F3** — só F1. O site oficial migrou para a
plataforma FOM (mesma do formula1.com, Next.js): `/Calendar` redireciona para
`/en/racing/{ano}`. O que o scraper usa, tudo 1ª parte:

1. **`fiaformula2.com/en/racing/{ano}`** — cards do calendário no payload React
   (`self.__next_f`) embutido no HTML: round + URL de cada etapa. Atenção: por
   causa do streaming do RSC, o link de alguns cards vem em um nó separado no
   fim do payload (ligado pelo id `_S_{n}_`).
2. **`fiaformula2.com/en/racing/{ano}/{local}`** — objeto `meeting` em JSON puro
   dentro do payload: sessões (`p`, `q`, `r1` sprint, `r` feature) com horário
   local + `gmtOffset`, estado, dados do circuito e resultados da sessão padrão.
3. **`api.formula1.com/v2/core-fom-results/f2/race?meeting=…&session=…`** — API
   do próprio site para os resultados que não vêm embutidos (a página só embute
   os da feature). Usa a chave **pública** servida no HTML a todo visitante
   (extraída dinamicamente do payload, caso rotacione). Os demais endpoints
   (`core-editorial-races/...`) são restritos a chaves privadas — não usar.
- **Limitação**: o site só publica a **temporada corrente**; para temporadas
  passadas a fonte seria a Wikipedia (não implementado).
- ❌ Agregadores/terceiros privados: **não usar**.

## Estrutura do fim de semana → `event_sessions`

Formato F2 atual:
- `practice` — treino livre (1).
- `qualifying` — define o grid.
- **`sprint_race`** — corrida sprint (sábado), grid parcialmente invertido.
- **`feature_race`** — corrida principal (domingo), com pit stop obrigatório.

## Mapeamento para o schema

- **Championship**: `f2`.
- **Season**: `year`.
- **Circuit**: **reutilizar os da F1** (Melbourne, Bahrein, Monza, Spa…) por
  `slug`. Só criar novos se houver venue exclusivo.
- **Event**: round no calendário; `external_id = f2-{ano}-{round}`.
- **Result**: dois vencedores por fim de semana (sprint + feature). Como são
  sessões distintas, o vencedor pode ser guardado por **sessão** ou usar
  `event_results` com `class` = `sprint`/`feature` (aproveitando a coluna
  multi-classe do schema).

## Particularidades / desafios

- **Duas corridas por evento** (sprint + feature): o modelo "1 resultado por
  evento" não basta → usar `event_results.class` (`sprint`/`feature`) ou
  vincular resultado à sessão.
- **Sem API estável garantida**: o scraper de HTML precisa ser resiliente a
  mudanças de layout; a API da FIA deve ser preferida se funcionar.
- **Datas atreladas à F1**: pode-se derivar parte do calendário do próprio
  evento de F1 correspondente.

### Frontend
- Exibir **duas corridas** por evento (sprint + feature) com resultados próprios.
